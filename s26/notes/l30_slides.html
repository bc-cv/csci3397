SECTION 2 · 12 min

LLaVA: A Generative Image-Text Model

Large Language and Vision Assistant
2 · LLaVA Motivation

The limit of CLIP

  • CLIP can match images to texts, but it cannot generate text
  • It cannot answer free-form questions, hold a conversation, or reason step by step
  • What we want: a model that sees and talks
Idea: bolt a vision encoder onto a pretrained LLM. Reuse all the language reasoning we already have.
Lec. 30 · Multimodal FMs15
2 · LLaVA Training

Two-stage post-training

Stage 1 — Feature alignment

  • Train only the projector
  • Vision encoder + LLM frozen
  • Data: image–caption pairs
  • Goal: map visual features into the LLM's token space

Stage 2 — Instruction tuning

  • Fine-tune projector + LLM
  • Data: multimodal instruction-following examples (visual Q&A, reasoning)
  • Goal: respond like a helpful visual assistant
Key insight: a frozen LLM + a small projector is enough to inherit language reasoning for visual inputs — extremely cheap to train.
Lec. 30 · Multimodal FMs17
2 · LLaVA Capabilities

What LLaVA can do

  • Describe an image in natural language
  • Answer questions about image content
  • Reason about images: "why is this funny?", "what should I do next?"
  • Hold a multi-turn conversation grounded in an image

SECTION 3 · 15 min

The Design Space of Multimodal Foundation Models

Five dimensions that shape every MFM
3 · DESIGN SPACES Roadmap

Before we dive into models...

Every MFM is a point in a 5-dimensional design space. Understanding the dimensions lets us compare any two models — and reason about new ones.

  1. Input / Modalities — what goes in?
  2. Method / Architecture — how are modalities combined?
  3. Output / Task — what comes out?
  4. Pretraining / Supervision — how is it trained?
  5. Prompt / Instruction — how do users interact?
Today's models — CLIP LLaVA — pick very different points in this space.

3.2 · METHOD / ARCHITECTURE How are modalities combined?

Five fusion strategies

ApproachDescriptionExamples
Separate encoders + fusionEach modality has its own encoder; fused via cross-attention or concatenationFlamingo, BLIP-2
Shared encoder (early fusion)Modalities mapped into a shared embedding space at the encoder levelCLIP, ALIGN
Encoder-decoder (Seq2Seq)Encode modalities, then autoregressively decode outputsLLaVA, VideoChat
Cross-attention transformersOne modality is the query, another is key/valueFlamingo, Kosmos-2
Mixture-of-modality expertsExperts specialize per modality; gating fuses outputsGato

3.3 · OUTPUT / TASK What comes out?

Output determines the model class

  • Discriminative outputs — classification, retrieval, similarity scores
  • Generative outputs — text, images, video, 3D, control/action
  • Structured outputs — segmentation masks, bounding boxes, keypoints
Many MFMs are promptable, just like LLMs:
  • Text query → image / video / 3D reasoning
  • Image → caption → answer / action
  • Multimodal instructions → robotic task
Lec. 30 · Multimodal FMs6
3.4 · PRETRAINING / SUPERVISION How is it trained?

Pretraining objectives

Self-supervised

  • Masked modeling (predict hidden tokens)
  • Contrastive alignment (pull matching pairs together)
  • Reconstruction (autoencoders, MAE)

Supervised

  • Paired datasets (image–caption, image–QA)
  • Instruction tuning
  • RLHF on multimodal preferences
Modern MFMs typically mix objectives: contrastive + generative + retrieval, often in stages.
Lec. 30 · Multimodal FMs7
3.5 · PROMPT / INSTRUCTION How do users interact?

Prompting modalities

  • Text prompts — natural language queries (most common)
  • Visual prompts — points, boxes, masks (Lec. 29 — SAM)
  • Multimodal prompts — text + image together (e.g., "describe what's wrong with this X-ray")
  • Structured prompts — JSON schemas, function calls, tool use
Key idea: the prompt is the interface. Same model, different prompts → different tasks.

3 · DESIGN SPACES Today's focus

Two image-text MFMs that bracket the design space

CLIP Discriminative

  • Dual encoder, no fusion
  • Contrastive pretraining
  • Output: similarity score
  • Use: zero-shot classification, retrieval

LLaVA Generative

  • Encoder + projector + LLM
  • Instruction tuning
  • Output: free-form text
  • Use: VQA, conversation, reasoning

Same modalities, very different design choices → very different capabilities.

5 · WRAP-UP Key takeaway

One framework, many points

  • Every MFM is a point in a 5-dimensional design space: modalities × architecture × output × pretraining × prompting
  • CLIP and LLaVA show two very different points — discriminative vs. generative — built from the same image-text ingredients
  • Design choices, not modalities, decide what a model can do
